# `kubernetes/` — Flux-managed cluster manifests

This directory is the source of truth for the Tier-1 k3s cluster. Every
manifest here is reconciled by Flux v2 from `git.psimaker.org/umut.erdem/homelab`,
branch `main`, path `kubernetes/`.

For the *why* behind the choices below, see
[`docs/architecture.md`](../docs/architecture.md). For the *how* of running a
single workload, see the per-directory READMEs.

## Layout

```
kubernetes/
├── README.md                 (this file)
├── flux-system/              Flux GitRepository + parent Kustomizations
│   ├── gotk-components.yaml  Generated by `flux bootstrap` — see scripts/bootstrap.sh
│   ├── gotk-sync.yaml        GitRepository + infrastructure + apps Kustomizations
│   └── kustomization.yaml    Aggregates the two
├── infrastructure/           Cluster controllers, storage, observability, identity
│   ├── controllers/          Cilium, Traefik, cert-manager, ESO, cloudflared, kube-vip, Renovate
│   ├── storage/              Longhorn
│   ├── observability/        kube-prometheus-stack, Loki, Promtail, Tempo, Beszel, alerts
│   └── identity/             Pocket-ID, Tinyauth, Headscale
└── apps/                     User-facing workloads (LOOGI etc.) — populated separately
```

## Flux topology

```
flux-system Kustomization (./kubernetes)
├── infrastructure  Kustomization (./kubernetes/infrastructure, interval=10m, wait=true)
│   ├── controllers/...
│   ├── storage/longhorn
│   ├── observability/...
│   └── identity/...
└── apps            Kustomization (./kubernetes/apps,            interval=1m,  dependsOn=infrastructure)
```

The two child Kustomizations have different intervals on purpose: controllers
reconcile every 10 minutes (they rarely change, and Helm release reconciles are
costly), apps every 1 minute (workloads iterate quickly). HelmReleases inherit
their own 5-minute reconcile.

## Conventions

| Convention                     | Why                                                  |
| ------------------------------ | ---------------------------------------------------- |
| One namespace per directory    | `kustomize build` works without name collisions      |
| HelmRelease pins chart version | Renovate opens PRs; nothing floats                   |
| Container images pinned by tag (and digest where supported) | Reproducible rollbacks                |
| Renovate `# renovate:` comments above every pinned version  | Custom regex manager picks them up    |
| Secrets are `*.sops.yaml` only | Plaintext blocked by pre-commit and CI               |
| `valuesFrom: ConfigMapRef` over inline `values:` for Helm   | Renovate can scan helm-values files   |
| Labels: `app.kubernetes.io/managed-by: flux` everywhere     | Quick filter when triaging            |

## Bootstrapping

From an empty cluster:

```bash
# 1. Provision: see scripts/bootstrap.sh — Hetzner + DNS + nodes
# 2. Pre-seed sops-age secret (required so Flux can decrypt during reconcile)
kubectl create namespace flux-system
kubectl create secret generic sops-age \
  --namespace=flux-system \
  --from-file=age.agekey=$HOME/.config/sops/age/keys.txt

# 3. Bootstrap Flux (this writes gotk-components.yaml + gotk-sync.yaml)
flux bootstrap gitea \
  --hostname=git.psimaker.org \
  --owner=umut.erdem \
  --repository=homelab \
  --branch=main \
  --path=kubernetes \
  --version=v2.4.0
```

After bootstrap, `flux get kustomizations` should show both
`flux-system/infrastructure` and `flux-system/apps` as `Ready=True`.

## Verifying changes locally

```bash
# Lint
yamllint kubernetes/
kubeconform -summary -ignore-missing-schemas kubernetes/

# Render with kustomize
kustomize build kubernetes/infrastructure | less

# Verify SOPS-encrypted files only contain encrypted blobs
find kubernetes -name '*.sops.yaml' -exec sops -d {} > /dev/null \;
```

These checks run in `.gitea/workflows/lint.yml` on every PR.
